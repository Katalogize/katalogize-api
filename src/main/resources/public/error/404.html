Katalogize API is the backend for Katalogize, a web application to create catalogs about anything you want and however you need with customizable templates.

Visit Katalogize

Purpose

The purpose of the public Katalogize API is to enable devs that want to display catalogs data in their aplications in a simple way with GraphQL.

This way, you are able to retrieve the exact data you need from your favorite public catalogs and use it however you would like in your application.

How to use

We recommend only using the endpoints for retrieving catalog data, since any other endpoint would require authentication and should only be used on Katalogize App.

To retrieve public catalog data, request a POST to the https://api.katalogize.com/graphql endpoint with the following GraphQL query:

	
query {
  getCatalogByUsernameAndCatalogName(username: "User", catalogName:"Name") {
    name,
    description,
    items {
      name
      fields {
        fieldType: __typename
        ... on ItemFieldNumber {
          order,
          name,
          numberValue: value
        }
        ... on ItemFieldString {
          order,
          name,
          stringValue: value
        }
        ... on ItemFieldImage {
          order,
          name,
          imageValue: value {
            path
          }
        }
      }
    }
  }
}

You can also use the exact same properties for queries like: getCatalogsByUsername and getOfficialCatalogs.
